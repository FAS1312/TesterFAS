Тест план по проверке создания задач с приоритетом и редактированию их

Объект тестирования:
    Вэб-приложение todolist
Функции объекта:
    Создание задач, удаление задач, редактирование задач, задание приоритета, задание выполнения
Проверка функций:
    Создание задач с приоритетом, редактирование приоритета у задачи

Стратегия:
    1. Тестирование серверной части
        1.1 Проверка API и запросов по url localhost:8000/api/tasks/
        1.2 Проверка API и запросов по url localhost:8000/api/tasks/<id>
    2. Тестирование клиентской части
        2.1 Проверка сохранения приоритета по умолчанию при создании задачи
        2.2 Проверка сохранения задачи с высоким приоритетом при создании задачи
        2.3 Проверка сохранения задачи с низким приоритетом при создании задачи
        2.4 Проверка изменения приоритета задачи со среднего на высокий
        2.5 Проверка изменения приоритета задачи со среднего на низкий
        2.6 Проверка изменения приоритета задачи с низкого на средний
        2.7 Проверка изменения приоритета задачи с низкого на Высокий
        2.8 Проверка изменения приоритета задачи с Высокого на Низкий
        2.9 Проверка изменения приоритета задачи с Высокого на Средний

Критерий начала:
    Готовы методы API для get и post запросов по url localhost:8000/api/tasks/, и для get, put и delete запросов по url localhost:8000/api/tasks/<id>
Критерий окончания:
    Результаты всех тесткейсов Pass

План тестового прогона:
    1. Открыть тесткейсы
    2. Для серверной части открыть localhost:8000, для клиентской localhost:1234
    3. Выполнить отправку соответсвтующих запросов по тесткейсам для серверной части
    4. Выполнить нужный путь для получения ожидаемых результатов для клиентской части
    5. Сделать заключение по прохождению тестов
